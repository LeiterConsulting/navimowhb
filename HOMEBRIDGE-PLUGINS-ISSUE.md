# Homebridge Plugins Issue Draft

Use this text when opening a Homebridge plugins issue for icon handling after the plugin is eligible for that workflow.

## Suggested Title

`Icon request for homebridge-navimow`

## Suggested Body

```md
Plugin name: `homebridge-navimow`

GitHub repository: `https://github.com/LeiterConsulting/navimowhb`

npm package: `https://www.npmjs.com/package/homebridge-navimow`

Current published version: `0.1.1`

Requested action: add the attached Navimow icon for the Homebridge Plugins page.

Brand asset links:

- Square icon (preferred source): `https://raw.githubusercontent.com/LeiterConsulting/navimowhb/main/brand/navimow-square.svg`
- Wide banner: `https://raw.githubusercontent.com/LeiterConsulting/navimowhb/main/brand/navimow-banner.svg`
- Social card: `https://raw.githubusercontent.com/LeiterConsulting/navimowhb/main/brand/navimow-social-card.svg`

Notes:

- The plugin already ships its own custom UI branding inside `homebridge-ui/public/`.
- We understand the main Homebridge Plugins page icon is managed separately from package assets.
- If you need the icon in a different format or size, I can provide it from the same artwork set.
```

## Submitter Checklist

- confirm the package name and published version are current
- confirm the raw GitHub asset links resolve from the default branch
- attach or link the square icon first unless the Homebridge team asks for another format
- if verification is still pending, wait until the plugin is eligible for the icon workflow before filing
